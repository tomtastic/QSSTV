Fix build #30: Use reinterpret_cast for mainWindowPtr

Build #30 failed because static_cast requires the full type definition
to verify inheritance relationships, but mainWindow is only forward-declared
in appglobal.h.

Changed from static_cast to reinterpret_cast in wavio.cpp:
- Line 76: dirDialog d(reinterpret_cast<QWidget*>(mainWindowPtr), ...)
- Line 208: dirDialog d(reinterpret_cast<QWidget*>(mainWindowPtr), ...)

reinterpret_cast doesn't require the full type definition and is appropriate
here since we know at runtime that mainWindow inherits from QWidget, even
though the compiler can't verify it at this point in the translation unit.

This is the correct solution for the forward-declaration scenario.
