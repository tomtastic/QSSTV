Pass large objects by const reference for performance

Medium priority performance improvements:

Changed function signatures to pass QString, QImage, and QByteArray
by const reference instead of by value. This avoids unnecessary copies
and reduces memory allocations.

Modified headers:
- textdisplay.h: append(const QString &)
- extviewer.h: setup(const QString &)
- vumeter.h: setLabelText(const QString &)
- basegraphicitem.h: setImage(const QImage &)
- waterfalltext.h: setText/getDuration(const QString &)
- soundbase.h: errorHandler(const QString &, const QString &)
- imageviewer.h: setParam/save(const QString &)
- rxwidget.h: setDRMStatusText/setOnlineStatus/setSSTVStatusText(const QString &)
- txwidget.h: setImage/setPreviewWidget/startNotifyCheck/sendHybrid(const QString &)
- txwidget.h: setDRMNotifyText/appendDRMNotifyText(const QString &)
- gallerywidget.h: putRxImage(const QString &)
- arraydumper.h: All arrayDump functions(const QString &)
- supportfunctions.h: setValue(const QString &)

Benefits:
- Eliminates deep copies of QString objects (reference counted but still overhead)
- Reduces memory allocations and deallocations
- Improves performance in frequently called functions
- No change to calling code required (implicit conversion works)
