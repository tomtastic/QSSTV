Fix build #31: Create non-const copy for openImage call

Build #31 failed because extviewer.cpp passes a const QString & to
imageViewer::openImage(), which expects a non-const QString & reference
(because it may modify the filename).

Fixed in extviewer.cpp setup():
- Create a non-const copy of the filename parameter
- Pass the copy to openImage() instead of the const reference

This resolves the const-correctness mismatch while maintaining the
performance benefits of passing by const reference to setup().
