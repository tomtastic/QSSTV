Fix build failure: Update .cpp implementations to match header signatures

This fixes build #27 failure caused by mismatched function signatures
between header and implementation files.

When we changed headers to use const QString & parameters, we forgot
to update the corresponding .cpp implementations, causing compilation
errors:
- error: out-of-line definition does not match any declaration

Fixed implementations in:
- soundbase.cpp: errorHandler(const QString &, const QString &)
- vumeter.cpp: setLabelText(const QString &)
- textdisplay.cpp: append(const QString &)
- extviewer.cpp: setup(const QString &)
- waterfalltext.cpp: setText/getDuration(const QString &)
- imageviewer.cpp: save/setParam(const QString &)
- rxwidget.cpp: setSSTVStatusText/setOnlineStatus(const QString &)
- txwidget.cpp: setImage/setPreviewWidget/startNotifyCheck(const QString &)
- gallerywidget.cpp: putRxImage(const QString &)
- arraydumper.cpp: All 6 arrayDump overloads(const QString &)
- supportfunctions.cpp: setValue(const QString &)

This completes the pass-by-const-reference refactoring from commit d73df5c.
