Fix wavio.cpp dirDialog constructor calls

Fixes remaining build errors from C-style cast removal in commit a44bee3.

The issue: mainWindowPtr is only forward-declared in appglobal.h, not
fully defined. When we removed the (QWidget*) cast, the compiler couldn't
verify that mainWindow* is convertible to QWidget*.

Solution: Use static_cast<QWidget*> instead of removing the cast entirely.
This provides type safety while still allowing the conversion.

Changed in wavio.cpp:
- Line 76: dirDialog d(static_cast<QWidget*>(mainWindowPtr), "Wave file")
- Line 208: dirDialog d(static_cast<QWidget*>(mainWindowPtr), "wave IO")

This completes the C-style cast modernization from commit a44bee3.
